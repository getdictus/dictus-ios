Create `tools/dict_builder.py` -- a Python 3.9+ script that:

1. Reads a frequency JSON file (format: `{"word": count, ...}`) from command line arg.
2. Builds an in-memory patricia trie from all words (lowercase, UTF-16 code units internally).
3. Serializes to the binary .dict format:

**Header (32 bytes):**
- magic: b"DTRI" (4 bytes)
- version: uint16 LE = 1
- flags: uint16 LE = 0
- node_count: uint32 LE
- word_count: uint32 LE
- max_freq: uint32 LE (max frequency in source data, used for normalization)
- reserved: 12 zero bytes

**Each trie node (variable length, children stored as sibling list):**
- flags: 1 byte
  - bits 7-6: children pointer size (00=no children, 01=2 bytes, 10=3 bytes, 11=4 bytes)
  - bit 5: has multiple characters (patricia compressed)
  - bit 4: is terminal (word end)
  - bits 3-0: reserved (0)
- char_count: 1 byte (only present if bit 5 set; otherwise implicit 1)
- characters: N * 2 bytes each as uint16 LE (UTF-16 code units)
- frequency: uint16 LE (only present if terminal). Normalize original frequency to 0-65535 range using log scale: `normalized = int(65535 * log(1 + freq) / log(1 + max_freq))`
- child_count: 1 byte (only present if children pointer size > 0) -- number of direct children
- children_offset: 2/3/4 bytes LE (absolute offset from file start to first child node; only present if children pointer size > 0)

**Node ordering:** Depth-first pre-order. Each node's children are stored contiguously as a sibling list, ordered alphabetically by first character. The children_offset points to the first child; child_count tells how many siblings to read.

**Patricia compression:** If a node has only one child and is not terminal, merge with child (concatenate characters). This reduces node count significantly for common prefixes.

4. Writes output .dict file.
5. Prints stats: word count, node count, file size, compression ratio.

**Usage:**
```bash
python3 tools/dict_builder.py DictusKeyboard/Resources/fr_frequency.json DictusKeyboard/Resources/fr_spellcheck.dict
python3 tools/dict_builder.py DictusKeyboard/Resources/en_frequency.json DictusKeyboard/Resources/en_spellcheck.dict
```

Create the C++ engine in `DictusKeyboard/Vendored/AOSPTrie/`. All files use `namespace dictus`, C++17, no Android/JNI dependencies, no dynamic allocation in hot paths.

**dictus_trie_format.h** -- Binary format constants:
- `DICTUS_MAGIC` = "DTRI"
- `DICTUS_VERSION` = 1
- `HEADER_SIZE` = 32
- Flag bit masks: `FLAG_CHILDREN_SIZE_MASK` = 0xC0, `FLAG_MULTI_CHAR` = 0x20, `FLAG_TERMINAL` = 0x10
- `struct DictusHeader` (packed, 32 bytes): magic[4], version(uint16), flags(uint16), node_count(uint32), word_count(uint32), max_freq(uint32), reserved[12]

**dictus_trie.h / dictus_trie.cpp** -- Trie reader:
- `class Trie`:
  - `bool loadMmap(const char* path)` -- open file, mmap with MAP_PRIVATE | PROT_READ, validate header magic="DTRI" and version=1, set madvise(MADV_RANDOM)
  - `void unload()` -- munmap + close fd
  - `~Trie()` calls unload()
  - `bool wordExists(const uint16_t* chars, int len) const` -- traverse trie matching characters, return true if terminal node reached
  - `uint16_t getFrequency(const uint16_t* chars, int len) const` -- same traversal, return frequency (0 if not found)
  - `const uint8_t* rootData() const` -- pointer to first node (data_ + HEADER_SIZE)
  - `uint32_t maxFreq() const` -- from header
  - Internal: `TrieNode readNode(uint32_t pos) const` -- read node at absolute offset, return struct with flags, characters, frequency, children_offset, child_count
  - Children iteration: given children_offset and child_count, read child_count consecutive nodes starting at that offset

**dictus_scorer.h / dictus_scorer.cpp** -- Weighted edit distance scorer:
- `struct Candidate { std::string word; float score; uint16_t frequency; }` (UTF-8 word for output)
- `class Scorer`:
  - `void setProximityMap(const ProximityMap* map)` -- set active proximity map
  - `std::vector<Candidate> correct(const Trie& trie, const uint16_t* input, int inputLen, float maxEditDist = 2.0f, int maxResults = 5) const`
  - Algorithm: recursive depth-first trie traversal. At each trie node, try all edit operations:
    - **Match**: if trie char == input char, cost 0, advance both
    - **Substitution**: if trie char != input char, cost = max(accentCost(input, trie), proximityCost(input, trie)). If accentCost returns >= 0, use it; else use proximity cost (minimum 0.4 for non-adjacent keys, as low as 0.2 for adjacent keys).
    - **Insertion** (extra char in input): cost 1.0, advance input only
    - **Deletion** (missing char in input): cost 1.0, advance trie only
    - **Transposition**: if next input char matches current trie char and current input char matches next trie char, cost 0.7, advance both by 2
  - Pruning: if accumulated cost > maxEditDist, prune branch (return immediately)
  - Scoring at terminal nodes: `score = (frequency / maxFreq) * (1.0 - accumulated_cost / maxEditDist)`
  - Results sorted by score descending, return top maxResults

- `float accentCost(uint16_t from, uint16_t to)` -- static function:
  - Base-to-accent pairs (return 0.15): e(0x65)<->e-acute(0xE9)/e-grave(0xE8)/e-circumflex(0xEA)/e-diaeresis(0xEB), a(0x61)<->a-grave(0xE0)/a-circumflex(0xE2), u(0x75)<->u-grave(0xF9)/u-circumflex(0xFB), o(0x6F)<->o-circumflex(0xF4), i(0x69)<->i-circumflex(0xEE)/i-diaeresis(0xEF), c(0x63)<->c-cedilla(0xE7)
  - Accent-to-different-accent of same base: 0.2
  - Not an accent pair: return -1.0 (caller uses proximity cost instead)

**dictus_proximity.h / dictus_proximity.cpp** -- Proximity maps:
- `class ProximityMap`:
  - `float distances_[26][26]` -- precomputed cost matrix for a-z pairs
  - `void buildAZERTY()` -- compute distances using AZERTY key positions:
    Row 0 (y=0): a(0.0) z(1.0) e(2.0) r(3.0) t(4.0) y(5.0) u(6.0) i(7.0) o(8.0) p(9.0)
    Row 1 (y=1): q(0.25) s(1.25) d(2.25) f(3.25) g(4.25) h(5.25) j(6.25) k(7.25) l(8.25) m(9.25)
    Row 2 (y=2): w(0.75) x(1.75) c(2.75) v(3.75) b(4.75) n(5.75)
    distance = min(1.0, euclidean(a,b) / 2.5)
  - `void buildQWERTY()` -- same with QWERTY positions:
    Row 0: q w e r t y u i o p
    Row 1: a(0.25) s(1.25) d(2.25) f(3.25) g(4.25) h(5.25) j(6.25) k(7.25) l(8.25)
    Row 2: z(0.75) x(1.75) c(2.75) v(3.75) b(4.75) n(5.75) m(6.75)
  - `float cost(uint16_t a, uint16_t b) const` -- if both a-z, return distances_[a-'a'][b-'a']; else 1.0

No `#include <iostream>` or `<string>` in headers -- keep headers minimal. Use `<cstdint>`, `<cstddef>`, `<vector>` only where needed.
