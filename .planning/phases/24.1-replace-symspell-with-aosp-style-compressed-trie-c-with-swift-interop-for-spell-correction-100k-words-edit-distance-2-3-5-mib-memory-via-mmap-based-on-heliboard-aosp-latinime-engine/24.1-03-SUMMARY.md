## Decisions Made
- **Two-pass user dictionary lookup**: Learned words are checked before trie to avoid false corrections on user-added words. This keeps user dictionary support without modifying the read-only mmap'd trie.
- **BFS serialization**: The original DFS serialization caused child offsets to be wrong because children were not contiguous in the binary file. BFS ensures children of each node are laid out sequentially, matching the C++ reader's expectations.
- **root_child_count in header**: Added to the binary format so the C++ reader knows where root-level children end, enabling correct traversal from the root node.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed trie serialization order (DFS to BFS)**
- **Found during:** Task 2 checkpoint (device verification)
- **Issue:** dict_builder.py serialized nodes in DFS order, but the C++ trie reader expects children of each node to be contiguous in memory. DFS interleaves children of different parents, causing incorrect child offset resolution and segfaults on keystroke.
- **Fix:** Rewrote dict_builder.py serialization to use BFS (level-order) traversal. Added root_child_count to the binary header. Updated C++ traverse() and scorer::correct() to use root_child_count. Regenerated both .dict files. Added iterative offset convergence (prev/current iteration separation).
- **Files modified:** tools/dict_builder.py, DictusKeyboard/Vendored/AOSPTrie/src/dictus_trie.cpp, DictusKeyboard/Vendored/AOSPTrie/src/dictus_scorer.cpp, DictusKeyboard/Vendored/AOSPTrie/include/dictus_trie.h, DictusKeyboard/Vendored/AOSPTrie/include/dictus_trie_format.h, DictusKeyboard/Resources/fr_spellcheck.dict, DictusKeyboard/Resources/en_spellcheck.dict
