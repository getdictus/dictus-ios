**AnimatedMicButton.swift -- Eliminate asyncAfter race and add .requested handling:**

1. **Fix the disabled state** -- change `.disabled()` modifier to disable for ALL states except `.idle`, `.ready`, and `.failed`:
```swift
.disabled(status != .idle && status != .ready && status != .failed)
```
This prevents rapid taps during `.requested`, `.recording`, and `.transcribing` states.

2. **Add rapid tap rejection logging** -- when the button action fires but status is not tappable, log `PersistentLog.log(.rapidTapRejected)`. Note: with .disabled this should not fire, but add it as a guard inside the action closure as belt-and-suspenders.

3. **Replace asyncAfter with immediate @State reset** -- find the `DispatchQueue.main.asyncAfter(deadline: .now() + 0.3)` pattern (around line 185) used for the success flash. Replace it with:
   - Set `showSuccessFlash = true` immediately
   - Then `withAnimation(.easeOut(duration: 0.3)) { showSuccessFlash = false }` -- SwiftUI animates the transition without a timer race

4. **Fix animation stacking** -- in the handleStatusChange method (or equivalent onChange handler), ALWAYS reset @State animation properties to concrete values WITHOUT animation BEFORE starting new animations. This cancels any existing repeating animations:
```swift
// Reset all animation state (cancels stacked repeating animations)
pulseScale = 1.0
glowOpacity = 0.3
shimmerOffset = -1.0
// Then start new animation for the new state
```

5. **Handle .requested state** -- in the status change handler, `.requested` should show a static state (no pulse, no glow animation). The button should appear in a "waiting" visual -- similar to idle but with the disabled modifier preventing taps.

6. **Log every status transition** inside the onChange handler:
```swift
PersistentLog.log(.statusChanged(from: oldStatus.rawValue, to: newStatus.rawValue, source: "micButton"))
```

**DictationCoordinator.swift -- Fix startDictation guard:**

Update the guard at the top of `startDictation(fromURL:)` (around line 179) to also allow `.requested`:
```swift
guard status == .idle || status == .failed || status == .ready || status == .requested else { return }
```

WHY: The keyboard sets status to `.requested` before the app receives the URL scheme. When the app processes `dictus://dictate`, the status is already `.requested`. Without this fix, the guard rejects the start.

Also add a transcription timeout watchdog start point: after transcription begins (when status changes to `.transcribing`), this is where Plan 02 will add the timer. For now, just add a log:
```swift
PersistentLog.log(.statusChanged(from: oldStatus.rawValue, to: status.rawValue, source: "coordinator"))
```
at the point where `updateStatus()` is called, so every coordinator state change is logged.
