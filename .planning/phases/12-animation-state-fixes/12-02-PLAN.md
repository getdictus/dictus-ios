**KeyboardState.swift -- Periodic watchdog timer + forceResetToIdle:**

1. **Add `forceResetToIdle()` method** that resets ALL dictation-related state:
```swift
func forceResetToIdle() {
    dictationStatus = .idle
    waveformEnergy = []
    recordingElapsed = 0
    statusMessage = nil  // if exists
    // Write to App Group so app side sees the reset
    defaults.set(DictationStatus.idle.rawValue, forKey: SharedKeys.dictationStatus)
    defaults.synchronize()
    DarwinNotificationCenter.post(DarwinNotificationName.statusChanged)
}
```

2. **Add watchdog timer** -- a `private var watchdogTimer: Timer?` and a `private var lastWaveformUpdate: Date` property:
- `startWatchdog()`: Creates a repeating Timer at 1.0s interval. On each tick, checks if `dictationStatus` is `.recording` or `.transcribing`. If yes and `Date().timeIntervalSince(lastWaveformUpdate) > 5.0`, logs `.watchdogReset(source: "keyboard", staleState: dictationStatus.rawValue)` and calls `forceResetToIdle()`. Uses `[weak self]` to avoid retain cycles (CRITICAL -- see research pitfall 3).
- `stopWatchdog()`: Invalidates and nils the timer.
- Start the watchdog when `dictationStatus` changes to `.recording` (in the status change handler / refreshFromDefaults). Stop it when status becomes `.idle`, `.ready`, or `.failed`.
- Update `lastWaveformUpdate = Date()` whenever waveformEnergy is refreshed from App Group defaults.

3. **Add `deinit`** that calls `stopWatchdog()` to prevent timer leaks across keyboard extension reloads.

**KeyboardRootView.swift -- Reset-on-appear stale state detection:**

Add an `.onReceive` for the `.dictusKeyboardWillAppear` notification (NOT .onAppear -- see research pitfall 4):
```swift
.onReceive(NotificationCenter.default.publisher(for: .dictusKeyboardWillAppear)) { _ in
    // Check for stale state from a previous crashed/abandoned session
    let activeStates: [DictationStatus] = [.requested, .recording, .transcribing]
    if activeStates.contains(state.dictationStatus) {
        // Read fresh from App Group
        state.refreshFromDefaults()
        // If still stale after refresh, force reset
        if activeStates.contains(state.dictationStatus) {
            PersistentLog.log(.watchdogReset(source: "keyboardAppear", staleState: state.dictationStatus.rawValue))
            state.forceResetToIdle()
        }
    }
}
```

Place this on the outermost VStack in body, alongside the existing .onAppear.

**DictationCoordinator.swift -- Transcription timeout watchdog:**

1. **Add `private var transcriptionWatchdog: Timer?`** property.

2. **`startTranscriptionWatchdog()`**: Creates a NON-repeating timer at 30.0s. When it fires: checks if `status == .transcribing`, if yes logs `.watchdogReset(source: "appTranscription", staleState: "transcribing")` and calls `cancelDictation()`. Uses `[weak self]`.

3. **`stopTranscriptionWatchdog()`**: Invalidates and nils the timer.

4. **Wire it up**: Start the watchdog when status transitions to `.transcribing` (inside the transcription start flow). Stop it when transcription completes successfully or when status changes away from `.transcribing` (in `updateStatus()` or wherever status changes are processed). Also stop in `cancelDictation()` cleanup.

5. **Clean up in deinit** if DictationCoordinator has one, or add one.

IMPORTANT: All timer closures MUST use `[weak self]` and all timers MUST be invalidated in appropriate cleanup paths to prevent retain cycles and memory leaks in the keyboard extension.
