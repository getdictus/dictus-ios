Three fixes in this task:

**1. Accent uppercase fix (root cause: case lost in lastTypedChar)**

In `KeyboardView.swift`, `insertCharacter()` stores `lastTypedChar = char` — this preserves the original case (e.g., "A" stays "A"). The problem is in `AccentedCharacters.adaptiveKeyLabel()` which calls `lastChar?.lowercased()` unconditionally, losing the case info. And `AdaptiveAccentKey.displayChar` then uses `isShifted` (which is already false due to auto-unshift) to re-case the result.

Fix:
- In `AccentedCharacters.swift`, modify `adaptiveKeyLabel(afterTyping:)` to preserve the original case. Lookup the accent using lowercased key, but if the original character was uppercase, return the accent uppercased. New signature stays the same but behavior changes:
  ```swift
  public static func adaptiveKeyLabel(afterTyping lastChar: String?) -> String {
      guard let lastChar = lastChar else { return "'" }
      let lowered = lastChar.lowercased()
      if let accent = defaultAccents[lowered] {
          // Preserve original case: if user typed "A", return "À" not "à"
          return lastChar == lastChar.uppercased() && lastChar != lastChar.lowercased()
              ? accent.uppercased() : accent
      }
      return "'"
  }
  ```
- In `AdaptiveAccentKey.displayChar`, remove the `isShifted` ternary. The accent already has the correct case from `adaptiveKeyLabel`. Just return the base value directly:
  ```swift
  private var displayChar: String {
      AccentedCharacters.adaptiveKeyLabel(afterTyping: lastTypedChar)
  }
  ```
- Similarly in `startLongPressTimer()`, derive accent popup case from `lastTypedChar` case instead of `isShifted`:
  ```swift
  let isUppercase = lastTypedChar?.uppercased() == lastTypedChar && lastTypedChar?.lowercased() != lastTypedChar
  if isUppercase {
      accentOptions = accents.map { $0.uppercased() }
  } else {
      accentOptions = accents
  }
  ```

**2. Special key background color fix**

In `SpecialKeyButton.swift`, change ALL instances of `Color(.systemGray3)` used as special key backgrounds to `Color(.systemGray5)`. This is a lighter gray that matches Apple's native keyboard special key color. Affected views:
- `ShiftKey` (line ~43, the `.off` state background)
- `DeleteKey` (line ~95)
- `ReturnKey` (line ~281)
- `GlobeKey` (line ~300)
- `EmojiKey` (line ~327)
- `LayerSwitchKey` (line ~482)

**3. Shift/caps lock active state styling fix**

In `ShiftKey`, swap the active state colors to match Apple convention:
- Active state (shifted or capsLocked): use `Color(.systemGray5)` background (light) with `Color(.label)` foreground (dark arrow) — Apple shows white bg + black arrow
- Inactive state (.off): keep `Color(.systemGray5)` background with `Color(.label)` foreground

The current code has it backwards: `Color(.label)` as background (dark) and `Color(.systemBackground)` as foreground (light).

Change:
```swift
.fill(shiftState != .off
      ? Color(.systemBackground)  // was Color(.label) — now light bg when active
      : Color(.systemGray5))      // was Color(.systemGray3) — lighter gray
```
And foreground:
```swift
.foregroundColor(shiftState != .off
                 ? Color(.label)              // was Color(.systemBackground) — now dark icon when active
                 : Color(.label))
```

Since foreground is Color(.label) in both states, simplify to just `.foregroundColor(Color(.label))` with no conditional.
