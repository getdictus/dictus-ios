Both copies of BrandWaveform must be updated to use 30 bars with adaptive bar width via GeometryReader. The two files should be functionally identical after this change.

**Changes to both files:**

1. Set `private let barCount = 30` (keyboard currently has 40, app already has 30).
2. Remove the stored `barWidth` property entirely (keyboard has `let barWidth: CGFloat = 5`, app has `@ScaledMetric private var barWidth: CGFloat = 4`). Bar width will be computed from available space.
3. Define `private let barSpacing: CGFloat = 2` (consistent spacing).
4. Wrap the HStack in a GeometryReader that computes bar width:
```swift
var body: some View {
    GeometryReader { geometry in
        let totalSpacing = barSpacing * CGFloat(barCount - 1)
        let barWidth = max((geometry.size.width - totalSpacing) / CGFloat(barCount), 2)

        HStack(spacing: barSpacing) {
            ForEach(0..<barCount, id: \.self) { index in
                barView(index: index, barWidth: barWidth)
            }
        }
        .frame(maxWidth: .infinity, maxHeight: .infinity)
    }
    .frame(height: maxHeight)
    .animation(.easeOut(duration: 0.08), value: energyLevels)
}
```
5. Update `barView` to accept `barWidth` as a parameter instead of reading the stored property. The cornerRadius should use `barWidth / 2`.
6. Keep the animation duration at 0.08s (the keyboard value — snappier for real-time audio feedback). The app's 0.15s is too slow for live waveform.
7. Keep existing `energyForBar` and `colorForBar` methods unchanged.
8. Add a comment at the top of each file: `// IMPORTANT: This file is duplicated in DictusApp/Design/ and DictusKeyboard/Design/. Keep both copies in sync.`
9. Keep the #Preview blocks in the DictusApp copy (remove or keep as-is). The DictusKeyboard copy has no previews.

WHY adaptive width: User decision says "bar width adapts automatically to fit available space in each context." GeometryReader calculates the exact bar width based on container width, so the waveform fills the space whether it appears in the recording overlay (full keyboard width), the HomeView card (narrower), or the RecordingView (full screen).
