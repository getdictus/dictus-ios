**Step 1: Create dictus_ngram_format.h**

Create `DictusKeyboard/Vendored/AOSPTrie/include/dictus_ngram_format.h` with the NGRM binary format definition:

```
Header (32 bytes):
  magic[4]: "NGRM" (0x4E, 0x47, 0x52, 0x4D)
  version: uint16 = 1
  flags: uint16 = 0 (reserved)
  bigram_count: uint32
  trigram_count: uint32
  bigram_offset: uint32   // absolute byte offset to bigram section
  trigram_offset: uint32   // absolute byte offset to trigram section
  string_table_offset: uint32  // absolute byte offset to string table
  string_table_size: uint32    // total bytes in string table
```

Bigram section (sorted by key_hash ascending):
Each entry: `key_hash: uint32, result_count: uint8, results: [{word_offset: uint32, score: uint16}]`

Trigram section (same layout, sorted by key_hash ascending):
Each entry: `key_hash: uint32, result_count: uint8, results: [{word_offset: uint32, score: uint16}]`

String table: packed null-terminated UTF-8 strings (all predicted words).

Use FNV-1a 32-bit hash for keys:
- Bigram key: lowercase previous word
- Trigram key: "word1\0word2" (null-separated, both lowercase)

Maximum 8 results per entry (result_count capped at 8).

Define these as C structs and constants in the header file within `namespace dictus`. Include `NGRM_MAGIC`, `NGRM_VERSION`, `NGRM_MAX_RESULTS = 8`, and the header struct `NgramHeader`.

**Create dictus_ngram.h** in `DictusKeyboard/Vendored/AOSPTrie/include/`:

```cpp
#ifndef DICTUS_NGRAM_H
#define DICTUS_NGRAM_H

#include <cstdint>
#include <cstddef>
#include <vector>
#include <string>

namespace dictus {

/// Result from n-gram prediction: a word and its score.
struct NgramResult {
    std::string word;
    uint16_t score;
};

/// N-gram lookup engine using mmap'd binary data.
/// Supports bigram and trigram lookup with Stupid Backoff scoring.
class NgramEngine {
public:
    NgramEngine();
    ~NgramEngine();

    /// Load n-gram binary file via mmap.
    bool load(const char* path);

    /// Unload mmap'd data.
    void unload();

    /// Whether data is loaded.
    bool isLoaded() const;

    /// Predict top-N next words given one previous word (bigram lookup).
    /// Returns results sorted by score descending, up to maxResults.
    std::vector<NgramResult> predictAfterWord(const char* word, size_t maxResults) const;

    /// Predict top-N next words given two previous words (trigram + bigram backoff).
    /// Uses Stupid Backoff: try trigram first, then bigram with lambda=0.4 discount.
    /// Returns merged results sorted by score descending, up to maxResults.
    std::vector<NgramResult> predictAfterWords(const char* word1, const char* word2, size_t maxResults) const;

    /// Get n-gram score for a specific word following one previous word.
    /// Used for correction boosting. Returns 0 if no match.
    uint16_t bigramScore(const char* prevWord, const char* word) const;

    /// Get n-gram score for a specific word following two previous words.
    /// Returns 0 if no match.
    uint16_t trigramScore(const char* word1, const char* word2, const char* word) const;

private:
    uint8_t* data_;
    size_t dataSize_;
    bool loaded_;

    /// FNV-1a 32-bit hash (must match Python implementation exactly).
    static uint32_t fnv1a(const uint8_t* bytes, size_t len);

    /// Binary search for a key_hash in a sorted section.
    /// Returns pointer to the entry if found, nullptr otherwise.
    const uint8_t* findEntry(uint32_t keyHash, const uint8_t* section, uint32_t count) const;

    /// Parse results from an entry pointer into NgramResult vector.
    std::vector<NgramResult> parseResults(const uint8_t* entry, size_t maxResults) const;

    /// Get string from string table at given offset.
    const char* getString(uint32_t offset) const;
};

} // namespace dictus
#endif
```

**Create dictus_ngram.cpp** in `DictusKeyboard/Vendored/AOSPTrie/src/`:

Implement the NgramEngine class:

1. **load()**: Same mmap pattern as dictus_trie.cpp:
   - `open(path, O_RDONLY)` -> `fstat()` -> `mmap(nullptr, size, PROT_READ, MAP_PRIVATE, fd, 0)` -> `close(fd)`
   - Validate magic bytes == "NGRM" and version == 1
   - Store pointer to mmap'd data and size

2. **unload()**: `munmap(data_, dataSize_)`, reset state

3. **fnv1a()**: FNV-1a 32-bit hash matching Python exactly:
   ```cpp
   uint32_t NgramEngine::fnv1a(const uint8_t* bytes, size_t len) {
       uint32_t h = 0x811c9dc5;
       for (size_t i = 0; i < len; i++) {
           h ^= bytes[i];
           h *= 0x01000193;
       }
       return h;
   }
   ```

4. **findEntry()**: Binary search on sorted entries:
   - Each entry starts with uint32 key_hash, followed by uint8 result_count, then result_count * (uint32 word_offset + uint16 score)
   - Entry size is variable: 5 + result_count * 6 bytes
   - Since entries are variable-length, use a linear scan with early termination (entries are sorted, so stop when key_hash > target). For better performance, build a small index of entry offsets during load(). Store bigram_offsets_ and trigram_offsets_ as vectors of pointers to each entry start.
   - Alternative (simpler, good enough for <200K entries): during load(), build a vector of (key_hash, entry_pointer) pairs for bigrams and trigrams separately. Binary search on the key_hash in this index.

5. **predictAfterWord()**: Hash the lowercased previous word, binary search in bigram section, parse and return results.

6. **predictAfterWords()**: 
   - Hash "word1\0word2" (null byte separator), search trigram section
   - If trigram found: return trigram results
   - If no trigram match: fall back to bigram lookup for word2, apply Stupid Backoff discount (multiply scores by 0.4, i.e. `score * 4 / 10` in integer math)
   - If both found: merge trigram results with discounted bigram results, sort by score, return top maxResults

7. **bigramScore()**: Hash prevWord, find in bigram section, scan results for word match, return score or 0.

8. **trigramScore()**: Hash "word1\0word2", find in trigram section, scan results for word match, return score or 0.

9. **getString()**: Return `(const char*)(data_ + stringTableOffset + offset)` with bounds checking.

10. **Index building in load()**: After mmap, walk bigram and trigram sections to build index vectors. Each index entry is `{uint32_t hash, const uint8_t* ptr}`. This runs once at load time.

All string comparisons use lowercased input. Use a local lowercase buffer (stack-allocated char[256]) for incoming words before hashing.

Include headers: `<cstdint>`, `<cstring>`, `<cstdlib>`, `<algorithm>`, `<vector>`, `<sys/mman.h>`, `<sys/stat.h>`, `<fcntl.h>`, `<unistd.h>`, `"dictus_ngram_format.h"`, `"dictus_ngram.h"`
