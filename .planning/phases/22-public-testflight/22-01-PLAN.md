3. **Delete the `scrollToken` @State variable** (line 32). No more ScrollViewReader scrollTo needed.

4. **Add a new `currentCategoryEmojis` computed property** that returns only the selected category's emojis:
```swift
private var currentCategoryEmojis: [EmojiGridItem] {
    if selectedCategoryID == "recents" {
        return recentEmojis.enumerated().map { i, emoji in
            EmojiGridItem(id: "recents_\(i)", emoji: emoji, categoryID: "recents")
        }
    }
    guard let cat = categories.first(where: { $0.id == selectedCategoryID }) else {
        return []
    }
    return cat.emojis.enumerated().map { i, emoji in
        EmojiGridItem(id: "\(cat.id)_\(i)", emoji: emoji, categoryID: cat.id)
    }
}
```

5. **Rewrite the `normalMode` view builder** to remove ScrollViewReader and use a simple LazyHGrid with `currentCategoryEmojis` instead of `flatItems`:
```swift
@ViewBuilder
private var normalMode: some View {
    // Category name label
    HStack {
        Text(selectedCategoryName)
            .font(.system(size: 13, weight: .medium))
            .foregroundColor(.secondary)
        Spacer()
    }
    .padding(.horizontal, 8)
    .padding(.top, 4)
    .padding(.bottom, 1)

    // Single-category emoji grid
    ScrollView(.horizontal, showsIndicators: false) {
        LazyHGrid(rows: gridRows, alignment: .top, spacing: 0) {
            ForEach(currentCategoryEmojis) { item in
                Button {
                    HapticFeedback.keyTapped()
                    onEmojiInsert(item.emoji)
                    RecentEmojis.add(item.emoji)
                    recentEmojis = RecentEmojis.load()
                } label: {
                    Text(item.emoji)
                        .font(.system(size: 32))
                        .frame(width: emojiCellWidth, height: rowHeight)
                }
                .id(item.id)
            }
        }
        .padding(.horizontal, 2)
        .id(selectedCategoryID)  // Force grid rebuild on category change to release old glyphs
    }

    EmojiCategoryBar(
        sections: sectionInfos,
        selectedCategoryID: selectedCategoryID,
        onSelectCategory: { id in
            selectedCategoryID = id
        },
        onSearch: { isSearchActive = true },
        onDelete: onDelete,
        onDismiss: onDismiss
    )
}
```

Key changes in the grid:
- `.id(selectedCategoryID)` on the ScrollView forces SwiftUI to destroy and recreate the grid when the category changes, which releases the old Text views and their glyph caches
- After inserting an emoji, reload `recentEmojis` so the recents category stays current
- No more `scrollToken` or `onChange(of: scrollToken)` -- category bar directly sets `selectedCategoryID`
- No more `ScrollViewReader` wrapper

6. **Remove the `onChange(of: scrollToken)` block** entirely (lines 158-164 in current file).

7. **Keep search mode completely unchanged** -- it already uses a separate horizontal HStack with limited results, not the full grid. No memory issue there.

No functional changes needed. The `onSelectCategory` callback already passes the category ID. The bar already highlights the selected category.
