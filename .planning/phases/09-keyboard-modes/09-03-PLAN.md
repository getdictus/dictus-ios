  <action>
Create `DictusApp/Views/KeyboardModePicker.swift` — reusable component for Settings and onboarding (per locked decision).

**Structure:**
```
KeyboardModePicker
├── Segmented Picker ("Micro" | "Emoji+" | "Complet")
└── Miniature Preview (non-interactive, ~140pt height)
    ├── MicroModePreview: large circle (mic icon) centered, small globe icon bottom-left
    ├── EmojiModePreview: grid of colored rounded rects (emoji placeholders), toolbar bar at top with mic circle
    └── FullModePreview: 4 rows of small rounded rects (key placeholders), toolbar bar at top
```

**Parameters:**
- `@Binding var selectedMode: String` — raw value of KeyboardMode, bound to @AppStorage in parent

**Miniature previews (Claude's discretion on exact proportions):**
- Use simplified geometric shapes — rectangles for keys, circle for mic, colored blocks for emoji grid. Key labels at preview scale would be illegible.
- Wrap each preview in a rounded rectangle with `Color.dictusBackground` or similar dark background.
- Use `.allowsHitTesting(false)` on the preview area to make it non-interactive.
- Each preview ~140pt height, full width with horizontal padding.
- Use brand colors: dictusAccent for mic, gray for keys, varied pastels for emoji blocks.

**Segmented picker:**
- `Picker("Mode", selection: $selectedMode)` with `.pickerStyle(.segmented)`
- Tags: `KeyboardMode.micro.rawValue`, `KeyboardMode.emojiMicro.rawValue`, `KeyboardMode.full.rawValue`
- Labels use `KeyboardMode.displayName` values: "Micro", "Emoji+", "Complet"

Import SwiftUI and DictusCore.
