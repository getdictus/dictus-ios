  Rewrite EmojiMicroModeView.swift body. Keep the same struct signature. Changes:

  **1. Replace globe with gear icon (settings link):**
  Replace the globe Button in the toolbar HStack with a Link matching ToolbarView's pattern:
  ```swift
  Link(destination: URL(string: "dictus://")!) {
      Image(systemName: "gearshape.fill")
          .font(.system(size: 16, weight: .medium))
          .foregroundColor(Color(.systemGray))
          .frame(width: 32, height: 32)
  }
  ```
  WHY: The system already provides a globe icon. The gear icon opens the Dictus app for settings, matching the ToolbarView pattern. Users specifically requested this.

  **2. Wire onDismiss to advanceToNextInputMode:**
  Change the EmojiPickerView onDismiss from no-op to:
  ```swift
  onDismiss: {
      controller.advanceToNextInputMode()
  }
  ```
  WHY: In emojiMicro mode, the ABC button in EmojiCategoryBar should serve as the keyboard switcher (replacing the removed globe). Tapping ABC cycles to next input method.

  **3. Add .clipped() to prevent overflow:**
  Add `.clipped()` to the EmojiPickerView or its container to prevent the ScrollView from bleeding past the keyboard bounds horizontally and vertically.

  **4. Fix vertical space — increase total frame height:**
  The fundamental problem is that EmojiPickerView was designed to use the full keyboard height (in full mode, the toolbar is hidden when emoji is active). In emojiMicro, the 48pt toolbar steals space.

  Solution: Change the frame height from `totalHeight` to `totalHeight + 56` to match what full mode does when it expands for emoji (hides 48pt toolbar + 8pt spacer = 56pt extra). This gives EmojiPickerView the space it was designed for.

  The complete body should be:
  ```swift
  var body: some View {
      VStack(spacing: 0) {
          // Simplified toolbar: gear left, mic pill right.
          HStack {
              // Gear icon — opens Dictus app settings
              Link(destination: URL(string: "dictus://")!) {
                  Image(systemName: "gearshape.fill")
                      .font(.system(size: 16, weight: .medium))
                      .foregroundColor(Color(.systemGray))
                      .frame(width: 32, height: 32)
              }

              Spacer()

              // Mic pill for dictation
              AnimatedMicButton(status: dictationStatus, isPill: true, onTap: onMicTap)
          }
          .padding(.horizontal, 12)
          .frame(height: 48)

          // Emoji picker with clipping to prevent overflow.
          EmojiPickerView(
              onEmojiInsert: { emoji in
                  controller.textDocumentProxy.insertText(emoji)
              },
              onDelete: {
                  controller.textDocumentProxy.deleteBackward()
              },
              onDismiss: {
                  // ABC button switches to next input method (replaces removed globe).
                  controller.advanceToNextInputMode()
              }
          )
          .clipped()
      }
      .frame(height: totalHeight + 56)
      .clipped()
  }
  ```

  NOTE: The height increase (totalHeight + 56) will cause KeyboardViewController's heightConstraint to be slightly short. This is intentional — iOS will stretch the inputView to accommodate, and the extra height provides the space EmojiPickerView needs. If there are issues, the alternative is to pass a separate emojiTotalHeight from KeyboardRootView, but try this simpler approach first.
