**SuggestionBarView.swift** -- New file in DictusKeyboard/Views/:

Create a SwiftUI view displaying up to 3 suggestion slots in a horizontal row:
- Takes `suggestions: [String]`, `mode: SuggestionMode`, `onTap: (Int) -> Void` (index of tapped slot)
- Layout: HStack with thin vertical dividers (0.5pt width, Color(.systemGray3)) between slots
- Each slot is a Button with plain text, tapping calls `onTap(index)`
- Central slot (index 0 when 3 suggestions, or the first when fewer) is **bold** (`fontWeight(.semibold)`), lateral slots are regular weight

- Foreground color: `Color(.label)` for normal text
- No background on individual slots -- the toolbar background shows through
- If `suggestions` is empty, render `EmptyView()` (nothing shown)
- Transition: `.opacity` animation (0.15s) when suggestions appear/disappear

Word replacement helper (can be a private function in KeyboardRootView or a standalone utility):
```
func replaceCurrentWord(proxy: UITextDocumentProxy, currentWord: String, replacement: String, addSpace: Bool) {
    for _ in 0..<currentWord.count {
        proxy.deleteBackward()
    }
    proxy.insertText(replacement)
    if addSpace {
        proxy.insertText(" ")
    }
}
```

**KeyboardView.swift** -- Modify to accept and update SuggestionState:

Add a new parameter: `@ObservedObject var suggestionState: SuggestionState`

Wire suggestion updates on every keystroke:
1. In `insertCharacter(_ char:)`: after inserting the character, call `suggestionState.update(proxy: controller.textDocumentProxy)` via a `DispatchQueue.main.async` to let the proxy update first (pitfall 4 from research -- proxy reads can be stale immediately after insert).

2. In `onSpace` callback: BEFORE inserting the space, implement autocorrect:
   ```
   // Autocorrect flow:
   // a. Extract current word from proxy
   // b. If autocorrectEnabled and word is misspelled, get correction
   // c. Replace word with correction, store undo state
   // d. Then insert space
   // e. Clear suggestions
   ```
   - Extract current word using the same `documentContextBeforeInput` + `enumerateSubstrings(options: .byWords)` logic
   - Call `suggestionState.engine.spellCheck(currentWord)` (expose engine or add a `spellCheck` method on SuggestionState)
   - If correction found: call `replaceCurrentWord(proxy:currentWord:replacement:addSpace:false)`, then set `suggestionState.lastAutocorrect = AutocorrectState(originalWord: currentWord, correctedWord: correction, insertedSpace: true)`
   - Then insert space as normal
   - After space: `suggestionState.clear()`
   - Also clear on return/newline: `suggestionState.clear()`

3. In `onDelete` callback: BEFORE the normal deleteBackward, check for autocorrect undo:
   ```
   // Undo flow:
   // a. If lastAutocorrect exists:
   //    - Delete the corrected word (+ trailing space if insertedSpace)
   //    - Re-insert original word
   //    - Clear lastAutocorrect
   //    - Return (don't do normal delete)
   // b. Otherwise: normal deleteBackward, then update suggestions
   ```
   - After ANY action other than immediate backspace, clear `lastAutocorrect`: in `insertCharacter`, `onSpace`, `onReturn`, `onAccentAdaptive`, and suggestion tap handlers, set `suggestionState.lastAutocorrect = nil`.

4. In `onWordDelete` callback: clear `lastAutocorrect`, then after deletion call `suggestionState.update(proxy:)`.

5. Clear suggestions on layer switch (numbers/symbols/emoji): `suggestionState.clear()`
