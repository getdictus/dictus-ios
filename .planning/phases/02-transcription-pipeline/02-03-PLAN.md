**Step 1 — Create DictusApp/Models/ModelManager.swift:**
`@MainActor class ModelManager: ObservableObject` managing the full model lifecycle:

Properties:
- `@Published var downloadedModels: [String] = []` — identifiers of downloaded models
- `@Published var activeModel: String?` — currently selected model identifier
- `@Published var downloadProgress: [String: Float] = [:]` — per-model download progress (0.0-1.0)
- `@Published var modelStates: [String: ModelState] = [:]` — per-model state enum
- `private let defaults = AppGroup.defaults`

`ModelState` enum: `.notDownloaded`, `.downloading`, `.prewarming`, `.ready`, `.error(String)`

Methods:
- `func loadState()` — reads `SharedKeys.downloadedModels` (stored as JSON array of strings) and `SharedKeys.activeModel` from App Group defaults. Call on init.
- `func downloadModel(_ identifier: String) async throws` — uses `WhisperKit.download(variant:from:"argmaxinc/whisperkit-coreml", useBackgroundSession: false, progressCallback:)`. After download completes, prewarm via `WhisperKit(config)` with `prewarm: true, load: true, download: false`. Update `downloadedModels`, persist to App Group. Set `SharedKeys.modelReady = true` if this is the first model. Use foreground download for now (background session adds complexity — can iterate later if needed per RESEARCH open question #1).
- `func selectModel(_ identifier: String)` — sets `activeModel`, persists to `SharedKeys.activeModel`
- `func deleteModel(_ identifier: String) throws` — guard `downloadedModels.count > 1` (cannot delete last model). Remove model files from disk. Remove from `downloadedModels`. If deleted model was `activeModel`, switch to first remaining. Persist changes.
- `func isRecommended(_ identifier: String) -> Bool` — use `WhisperKit.recommendedModels()` to check if model matches device recommendation, or fallback to "small" as default recommendation.
- `private func persistState()` — write `downloadedModels` as JSON array and `activeModel` to App Group defaults, synchronize.

Model storage location: use `AppGroup.containerURL` + "/Models/" subdirectory. Pass this path as `modelFolder` to WhisperKit config. This keeps models in the shared container.

**Step 1 — Update TranscriptionService to accept model selection and apply filler filter:**

Modify `TranscriptionService.transcribe()` to:
1. Accept a `modelIdentifier: String` parameter (or have WhisperKit instance configured with the right model)
2. After getting raw transcription text from WhisperKit, apply `FillerWordFilter.clean(rawText)` before returning
3. Import DictusCore to access FillerWordFilter

The TranscriptionService should be able to load/switch WhisperKit instances for different models. Consider having a `func prepare(modelPath: String) async throws` that initializes WhisperKit with the specified model.

**Step 2 — Update DictationCoordinator to use SmartModelRouter:**

In `stopDictation()` (or wherever transcription is triggered after recording stops):
1. Calculate `audioDuration` from `audioRecorder.bufferSeconds`
2. Read `downloadedModels` from App Group defaults (or from ModelManager)
3. Call `SmartModelRouter.selectModel(audioDuration: audioDuration, downloadedModels: downloadedModels)`
4. Use the selected model identifier to configure TranscriptionService
5. Run transcription with the selected model

Import DictusCore to access SmartModelRouter.

**Step 3 — Ensure ModelManager and DictationCoordinator share WhisperKit state:**
The coordinator needs to know which models are downloaded and where they are stored. Options:
- Read `SharedKeys.downloadedModels` and `SharedKeys.activeModel` from App Group defaults directly
- Or hold a reference to ModelManager

Prefer reading from App Group defaults for loose coupling — the coordinator doesn't need to know about ModelManager's UI state.

**Step 4 — Handle edge cases:**
- No model downloaded: `startDictation()` should check `SharedKeys.modelReady` and fail with descriptive error ("No model downloaded. Open Dictus to download a model.") writing to `SharedKeys.lastError`
- Model loading failure: catch WhisperKit initialization errors, update status to `.failed`
- Transcription failure: catch and write error, don't crash
