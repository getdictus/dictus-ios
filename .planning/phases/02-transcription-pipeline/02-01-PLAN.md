**Step 2 — Create DictusApp/Audio/AudioRecorder.swift:**
`@MainActor class AudioRecorder: ObservableObject` that wraps WhisperKit's built-in AudioProcessor (do NOT build a custom AVAudioEngine pipeline — WhisperKit handles 16 kHz mono Float32 conversion internally):

```swift
import WhisperKit

@MainActor
class AudioRecorder: ObservableObject {
    private var whisperKit: WhisperKit?
    @Published var isRecording = false
    @Published var bufferEnergy: [Float] = []   // For waveform visualization
    @Published var bufferSeconds: Double = 0     // Elapsed recording time

    /// Initialize or re-use WhisperKit instance
    func prepare(whisperKit: WhisperKit) {
        self.whisperKit = whisperKit
    }

    func startRecording() throws {
        guard let whisperKit else { throw AudioRecorderError.notReady }
        // Configure AVAudioSession for recording
        let session = AVAudioSession.sharedInstance()
        try session.setCategory(.record, mode: .measurement, options: .duckOthers)
        try session.setActive(true)
        // Use WhisperKit's AudioProcessor — it handles format conversion
        try whisperKit.audioProcessor.startRecordingLive { [weak self] _ in
            DispatchQueue.main.async {
                guard let self, let wk = self.whisperKit else { return }
                self.bufferEnergy = wk.audioProcessor.relativeEnergy
                self.bufferSeconds = Double(wk.audioProcessor.audioSamples.count)
                    / Double(WhisperKit.sampleRate)
            }
        }
        isRecording = true
    }

    func stopRecording() -> [Float] {
        whisperKit?.audioProcessor.stopRecording()
        isRecording = false
        let samples = whisperKit?.audioProcessor.audioSamples ?? []
        // Deactivate audio session
        try? AVAudioSession.sharedInstance().setActive(false)
        return samples
    }
}
```

Handle `AVAudioSession.recordPermission` check: if `.undetermined`, request permission before starting. If `.denied`, set status to `.failed` with descriptive error.

- DecodingOptions: `task: .transcribe`, `language: "fr"`, `temperature: 0.0` (greedy, fastest), `usePrefillPrompt: true`, `usePrefillCache: true`, `skipSpecialTokens: true`

- Throws `TranscriptionError.notReady` if WhisperKit not initialized, `.emptyAudio` if samples array empty, `.transcriptionFailed(String)` for WhisperKit errors

Define error enums `AudioRecorderError` and `TranscriptionError` at file scope.

**Step 1 — Rewrite DictationCoordinator.swift:**
Replace the Phase 1 stub implementation with a real pipeline. Keep the existing class structure (`@MainActor`, `ObservableObject`, `static let shared`), `updateStatus()`, `writeTranscription()`, and `resetStatus()` methods. Replace `startDictation()` internals:

- Add properties: `private let audioRecorder = AudioRecorder()`, `private var transcriptionService: TranscriptionService?`, `@Published var bufferEnergy: [Float] = []`, `@Published var bufferSeconds: Double = 0`
- Add `WhisperKit` instance property, initialized lazily
- `startDictation()` flow:
  1. Cancel any in-flight dictation
  2. Check microphone permission (request if undetermined, fail if denied)
  3. Initialize WhisperKit if not already ready (use currently active model from App Group defaults — for now hardcode "openai_whisper-tiny" as fallback, Plan 2.3 will add model selection)
  4. `updateStatus(.recording)`
  5. Call `audioRecorder.startRecording()`
  6. Forward `audioRecorder.bufferEnergy` and `bufferSeconds` to coordinator's published properties (use Combine sink or direct observation)
- Add `stopDictation()`:
  1. `let samples = audioRecorder.stopRecording()`
  2. `updateStatus(.transcribing)`
  3. `let text = try await transcriptionService.transcribe(audioSamples: samples)`
  4. `writeTranscription(text)` — this already handles App Group write + Darwin notification
  5. `updateStatus(.ready)`
  6. Brief delay (0.5s) then signal "done" (for the checkmark flash before user returns to keyboard)
- Error handling: wrap in do/catch, on failure call `updateStatus(.failed)` and write error to `SharedKeys.lastError`

IMPORTANT: Preserve the existing pattern of writing ALL values to UserDefaults BEFORE posting Darwin notifications (race condition fix from Plan 1.4).
