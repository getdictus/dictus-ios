#### 1. App UI language switching (device English vs French)

**Test:** Build and run DictusApp on iPhone 17 Pro Simulator. Set device language to English. Navigate all screens: Home, Settings, Model Manager, Onboarding, Licenses, Sounds. Then change Simulator language to French (Settings > General > Language & Region > French) and repeat.
**Expected:** All UI labels, section headers, navigation titles, and button text switch languages automatically. French: "Reglages", "Langue de transcription", "Modeles", "Telecharges", etc. English: "Settings", "Transcription language", "Models", "Downloaded", etc.
**Why human:** String Catalog language resolution is a runtime iOS behavior — static file analysis confirms the catalog is correct but cannot confirm Xcode/SwiftUI resolves localizations at runtime.

#### 2. Keyboard extension language switching

**Test:** Enable the Dictus keyboard in a text field on the Simulator. Start a dictation to trigger RecordingOverlay. Also observe FullAccessBanner if Full Access is disabled.
**Expected:** Overlay text ("Starting...", "Transcribing...", "Listening...") and banner ("Full access required") display in the device language.
**Why human:** Keyboard extension targets have their own bundle and String Catalog lookup — cannot verify extension runtime behavior statically.

#### 3. Build validation (zero compilation errors)

**Test:** Open Dictus.xcodeproj and run Cmd+B for both DictusApp and DictusKeyboard targets.
**Expected:** Zero compilation errors. The new `String(localized:)` calls in ModelInfo+Localized.swift and all `Text("English string")` literals compile cleanly.
**Why human:** Requires Xcode Swift compiler; static grep cannot verify Swift type correctness or String Catalog key resolution.

#### 4. Transcription language picker does NOT control UI language

**Test:** With Simulator in English, open Settings > Transcription language > select "Français". Navigate all screens.
**Expected:** All UI remains in English. Only the WhisperKit/Parakeet engine language changes; UI language is not affected.
**Why human:** Behavioral separation of transcription language from UI language cannot be confirmed without running the app.
