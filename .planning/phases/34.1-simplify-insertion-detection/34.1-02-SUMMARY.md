## Decisions Made

- **Shrank `InsertionFailureReason` to 4 real cases.** Dropped `.deltaMismatch` and `.proxyDead` since Plan 34.1-01 proved the classifier never emits the corresponding outcomes. A stricter enum surface forces readers to understand only the 4 real failure modes the keyboard can actually report. The switch arm for `.deltaMismatch, .proxyDead` still exists inside the classifier-outcome switch (defensive fallback to `.success`), and that arm does not reference the deleted enum cases — it pattern-matches on `InsertionOutcome` which still carries them for source compat.
- **Dropped `attempts: Int` from `InsertTranscriptionResult.success`.** With single-shot execution, this field is always 1 and communicating "attempts" after removing the retry loop is misleading. Probe log still carries `attempt: 0` hardcoded per the plan for log-parser compatibility.
- **Defensive fallback for unreachable outcomes.** The switch arm handling `.deltaMismatch, .proxyDead` routes to `onComplete(.success(outcome:))` rather than `.failed(...)`. Rationale: if a future classifier regression accidentally re-introduces those outcomes, the degradation mode is "missing telemetry about an edge case" rather than "false-positive user-facing failure with lost text" — the former is a bug we fix later; the latter is a UX regression we don't want to ship.
- **Removed `escalateInsertionFailure` helper entirely.** The function had no remaining callers after the failure-branch simplification, plus its parameter list (`reason:`, `attempts:`) referenced the now-removed failure-enum cases. Deleting the function (not just its call site) prevents accidental resurrection.
- **Retained the 100ms App Group propagation retry in `handleTranscriptionReady`.** Plan action notes flagged this for review. It solves a different concern than insertText retries: cross-App-Group UserDefaults propagation can lag ~100ms on-device even after `synchronize()`. Removing it would regress the primary dictation path on cold-start races where the keyboard receives `transcriptionReady` before the App Group value has propagated. Kept as-is.
- **No LiveActivityManager references in KeyboardState.** The plan listed "LiveActivity transition to .failed" as one of three side-effects to remove. Grep confirmed no such reference in this file — the only `.failed` tokens in `KeyboardState.swift` are `DictationStatus.failed` comparisons. That item of the plan was already satisfied before this change.
