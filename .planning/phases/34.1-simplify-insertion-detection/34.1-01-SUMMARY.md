## Accomplishments

- `InsertionClassifier.classify(...)` rewritten with 7-rule success-first priority ordering per Phase 34.1 CONTEXT D1. Rule 1 (hasTextBefore=false → hasTextAfter=true) is authoritative regardless of nil context. Rule 4 treats negative delta with hasTextAfter=true as `.windowedSuccess` (iOS window truncation). Rule 6 narrows `.silentDrop` to the single true-failure window (delta=0, 0<beforeCount<400, hasText unchanged).
- Classifier structurally cannot emit `.proxyDead` or `.deltaMismatch` — verified by `grep` against the source and by the 17-test suite (zero assertions against those outcomes).
- Regression tests for all 3 real-device false-positive categories (empty-field nil-context, window-truncation negative delta, truncation-cap stuck delta) — each test documents the source probe line from the 2026-04-16 bug log and asserts the new success-family outcome.
- Existing Plan 34-01 tests that asserted `.proxyDead`/`.deltaMismatch` retargeted to the new policy (5 tests) — prevents stale expectations from breaking the suite and documents the policy migration inline.

## Decisions Made

- **Kept `.proxyDead` and `.deltaMismatch` enum cases as dead-but-retained.** A cleaner design would remove them, but InsertTranscriptionHelper (34-03) and LogEvent pattern-match on them with exhaustive switches. Removing cases would cascade into a helper rewrite that properly belongs to Plan 34.1-02 (retry loop removal). Keeping them prevents an otherwise-unavoidable Task 2 scope creep and is safe because tests guarantee the classifier never returns them.
- **Task 1 also pre-emptively retargeted existing `.proxyDead`/`.deltaMismatch` assertions** (not just added new tests). The plan's action notes instructed this conversion — doing it inside the RED commit keeps the whole test suite consistent with the new policy from the moment the implementation lands, rather than requiring a follow-up cleanup commit.
- **Used `DictusCore-Package` scheme from inside the DictusCore directory.** The plan's verification command (`-scheme DictusCore` at the project root) is no longer functional because `Dictus.xcodeproj/xcshareddata/xcschemes/` is untracked (currently showing as `??` in git status) and the root-level autogenerated `DictusCore` scheme contains no `TestAction`. Running from `DictusCore/` with the SPM-provided `DictusCore-Package` scheme exposes the test target correctly. Documented here so Plans 34.1-02/03 don't hit the same blocker.
