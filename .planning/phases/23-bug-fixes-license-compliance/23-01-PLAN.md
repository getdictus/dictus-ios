Two changes to invalidate autocorrect undo when the user types new characters or moves the cursor:

**Change 1 -- DictusKeyboardBridge.swift, in `handleInputKey(_ character: String)`:**
Add `suggestionState?.lastAutocorrect = nil` as the FIRST line inside the method body, BEFORE the `AudioServicesPlaySystemSound(KeySound.letter)` call. Add a comment explaining why:

```swift
private func handleInputKey(_ character: String) {
    // Invalidate autocorrect undo -- any new character input means the user
    // accepted the correction. Only immediate backspace should undo.
    suggestionState?.lastAutocorrect = nil

    AudioServicesPlaySystemSound(KeySound.letter)
    // ... rest unchanged
}
```

IMPORTANT: Place it BEFORE `insertText()` to prevent a fast backspace from racing with the character insertion.

**Change 2 -- KeyboardViewController.swift, in `textDidChange(_ textInput:)`:**
Add `bridge?.suggestionState?.lastAutocorrect = nil` between the `super.textDidChange(textInput)` call and the existing `bridge?.updateCapitalization()` call:

```swift
override func textDidChange(_ textInput: UITextInput?) {
    super.textDidChange(textInput)
    // Invalidate autocorrect undo on external text changes (paste, cursor tap, host autocorrect).
    bridge?.suggestionState?.lastAutocorrect = nil
    bridge?.updateCapitalization()
}
```

WHY textDidChange: This UIInputViewController callback fires when text changes externally. If the user taps elsewhere after autocorrect, undo context becomes invalid. UIInputViewController does NOT expose selectionDidChange, so textDidChange is the correct hook.
