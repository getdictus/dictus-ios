**Smooth overlay show/hide animation (DSGN-06):**

The recording overlay currently appears/disappears based on a conditional (`if showsOverlay` or `if state.dictationStatus != .ready`). Add transition animation so the overlay slides in and fades out smoothly.

Read KeyboardRootView.swift to understand the current conditional rendering structure. Then:

1. Add `.transition()` to the RecordingOverlay view:
   ```swift
   RecordingOverlay(...)
       .transition(.opacity.combined(with: .move(edge: .bottom)))
   ```

2. Add `.animation()` on the parent container that wraps the conditional:
   ```swift
   .animation(.easeOut(duration: 0.25), value: showsOverlay)
   ```
   Use whatever boolean or enum value drives the overlay visibility as the animation `value`.

3. If the current structure uses `if/else` without an explicit boolean, consider extracting a computed `showsOverlay` property for clearer animation binding.

**Important considerations from research:**
- SwiftUI won't animate `if/else` view swaps without explicit `.transition()` and `.animation()`
- The `.animation()` must be on the parent, not inside the conditional branches
- Verify that TimelineView (inside BrandWaveform) is fully removed from hierarchy when overlay is hidden (not just invisible) -- the `if/else` approach achieves this
- Apply the same easeOut animation to BOTH cancel dismiss AND transcription-complete dismiss (per user decision)

**Anti-pattern to avoid:** Do NOT use `DispatchQueue.main.asyncAfter` for animation timing. Use `withAnimation(.easeOut)` (established Phase 12 pattern).
