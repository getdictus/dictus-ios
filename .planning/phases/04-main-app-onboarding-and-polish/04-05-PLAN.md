    Fix 4 issues from keyboard UAT debug session:

    1. **KeyButton.swift — Key height and fixed font sizes:**
       - Change `keyHeight` from 42pt to 46pt (line ~217 in KeyMetrics) to match native iOS keyboard key height.
       - Replace `@ScaledMetric private var keyFontSize: CGFloat = 22` (line ~50) with `private let keyFontSize: CGFloat = 22`. This stops key labels from scaling with Dynamic Type — native iOS keyboard uses fixed key label sizes.
       - Replace `@ScaledMetric private var popupFontSize: CGFloat = 32` (line ~200 in KeyPopup) with `private let popupFontSize: CGFloat = 32`. Same reason.
       - Keep @ScaledMetric on any OTHER sizes (timer, icon sizes in toolbar) — only key labels should be fixed.

    2. **KeyboardViewController.swift — Update computeKeyboardHeight:**
       - After changing keyHeight to 46pt, the computed keyboard height changes. The formula is 4*keyHeight + 3*rowSpacing + verticalPadding + toolbarHeight. Verify the new total (~252-260pt). Update `computeKeyboardHeight()` if it uses hardcoded values; if it reads from KeyMetrics dynamically, no change needed.
       - The heightConstraint on the inputView must reflect the new total.

    3. **KeyboardRootView.swift — Fix layout shift on recording:**
       - The recording overlay must cover the FULL height of toolbar + keyboard (not just keyboard alone). When switching from KeyboardView to RecordingOverlay, the RecordingOverlay frame height should be `toolbarHeight + keyboardHeight` (i.e., the full VStack height).
       - Approach: Instead of replacing ToolbarView+KeyboardView with just RecordingOverlay, make RecordingOverlay's frame match the full content area. One way: keep the outer VStack height fixed (e.g., `.frame(height: totalHeight)`) and swap content inside, so the VStack never resizes. Or: set RecordingOverlay `.frame(height: keyboardHeight + toolbarHeight)`.
       - Ensure cancel/validate buttons have adequate top padding (at least 12pt) so they are not clipped.

    4. **FullAccessBanner.swift — Larger text, more padding, correct URL:**
       - Change font from `.dictusCaption` (~12pt) to `.dictusFootnote` or `.footnote` (~14pt) for both icon and text.
       - Increase vertical padding from 6pt to 10pt.
       - Increase the warning icon size (use `.font(.body)` or explicit size ~16pt).
       - Change the URL from `"app-settings:"` to `"dictus://settings"`. The DictusApp already has a `dictus://` URL scheme handler — add a `settings` path that the app can handle to show keyboard setup instructions. If the URL scheme path handling is not in scope, use `Link("Activer", destination: URL(string: "dictus://settings")!)` — the app will open even if it doesn't handle the specific path yet (better than opening a blank iOS Settings page).
