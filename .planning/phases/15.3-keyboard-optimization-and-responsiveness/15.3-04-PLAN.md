  <what-built>
Full keyboard optimization from issues #44 and #29:

1. Touch pipeline rewrite: Letter keys now use UIKit UIViewRepresentable (not SwiftUI DragGesture) for zero-latency touch handling. Audio and haptic fire on touchDown (not touchUp), matching Apple keyboard.

2. Key dimensions overhaul: Device-adaptive metrics across 3 device classes (SE/compact, standard, Plus/Max). Side margins now approximately equal inter-key gap. Row spacing increased to match Apple proportions.

3. Press feedback: Delete, space, return, globe, emoji, and layer switch keys now show background color change on press (brighter in dark mode, darker in light mode). Letter keys remain popup-only (no color change).

4. Performance infrastructure: Haptic enabled state cached (not read from UserDefaults per tap). Suggestion computation on background queue with coalescing. OSSignposter instrumentation ready for Instruments profiling.
  </what-built>
  <how-to-verify>
Open the Dictus keyboard in any text field on the iPhone 17 Pro simulator and verify:

1. Letter key press: Tap a letter key. Verify popup preview appears above the key. Verify NO background color change on the key itself.

2. Delete key press: Press and hold delete. Verify the key background changes to a lighter shade (dark mode) or darker shade (light mode) while held.

3. Space bar press: Tap the space bar. Verify background color change on press.

4. Return key press: Tap the return key. Verify background color change on press.

5. Accent long-press: Hold the "e" key for 400ms. Verify the accent popup appears. Slide finger to select an accent. Verify it inserts correctly.

6. Key spacing: Visually compare the keyboard layout with Apple's keyboard. Side margins should look proportional to inter-key gaps (not noticeably narrower). Row spacing should be noticeable between rows.

7. Typing flow: Type a sentence. Verify keys respond, characters insert, and suggestions appear. Verify no visual glitches or layout issues.

NOTE: Haptic and audio feedback cannot be verified on simulator. Physical device profiling with Instruments will be done separately.
  </how-to-verify>
