## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| T1: touchDown->highlight <= 16.67ms | #44 | Keyboard extension latency requires physical device + Instruments signpost profiling | Profile with Instruments > os_signpost, verify T1 interval <= 16.67ms |
| T2: touchDown->haptic <= 16.67ms | #44 | Haptic timing not measurable in tests | Profile with Instruments > os_signpost, verify T2 interval <= 16.67ms |
| T3: touchUp->insertText <= 33ms | #44 | Text insertion pipeline requires live keyboard | Profile with Instruments > os_signpost, verify T3 interval <= 33ms |
| No frame drops at 6-10 taps/s | #44 | Rendering performance requires real device GPU | Profile with Instruments > Animation Hitches, type rapidly |
| Key spacing matches Apple proportions | #29 | Visual comparison only | Screenshot Dictus vs Apple keyboard side by side on 3 device classes |
| Device-adaptive metrics | #29 | Requires multiple device form factors | Test on SE, standard, Plus simulators and compare |

**Justification:** Keyboard extension performance is inherently a physical device measurement. Signpost data is collected via Instruments. Layout verification requires visual comparison with Apple's keyboard. No unit tests can meaningfully validate "feels like native keyboard."
