<objective>
Fix keyboard waveform/overlay freeze when keyboard goes off-screen (#45 gap closure).

Purpose: iOS does NOT render SwiftUI views for off-screen keyboard extensions. When the keyboard goes off-screen (cold start app-switch dance, DI tap, app switching), SwiftUI stops body evaluation entirely. The existing renderTick + objectWillChange fixes (Plan 01) help when the extension process is suspended, but they do NOT solve the case where the keyboard view itself is off-screen -- SwiftUI never evaluates body regardless of state changes. On keyboard reappear, the overlay shows stale/frozen state.

Output: BrandWaveform recreated via .id() modifier on reappear with displayLevels seeded from current energy (not zeros), full overlay state refreshed from App Group, elapsed time recalculated.
</objective>
