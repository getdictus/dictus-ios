## Implementation Decisions

### Issue scope & priority
- **Bug fixes only** — no enhancements (#29, #31, #22) in this phase
- All 6 open bugs are ship-blockers for TestFlight beta
- Keyboard UX enhancements (#29 spacing, #31 settings shortcuts, #44 keyboard) deferred to Phase 15.3
- Auto-return (#23) stays deferred post-beta; close #9 as duplicate of #23
- Close #7 (logging) and #8 (CoreML pre-compile) as done — addressed in Phases 11/14

### Bug consolidation
- **Cluster 1 — Cold start overlay/waveform stability**: #37 is the parent issue (best root cause analysis). Close #16, #21, #36 referencing #37
  - Common root cause: cold start transitions between DictusApp and keyboard extension
  - Warm start works fine — the bugs are specific to app↔keyboard roundtrips during cold start
- **Cluster 2 — Dynamic Island lifecycle**: #42 (desync after chaining) and #41 (card not cleared via DI/lockscreen) are distinct issues with separate fixes

### Dynamic Island bugs
- **#37 fix approach**: Let Claude evaluate the 3 proposed fixes during research (not blindly apply). The fixes look well-analyzed but need validation against current codebase state
- **#42 fix approach**: Refactor LiveActivityManager with a strict state machine (standby→recording→processing→standby) that prevents invalid transitions — not just guards/debounce
- **#41 fix approach**: Unify all entry paths (mic button, DI tap, lockscreen notification) through the same reset routine before starting a new recording

### Verification
- Create a manual testing checklist for device verification (cold start, chaining recordings, DI tap, lockscreen, warm start regression)
- No automated tests in this phase

### Code cleanup scope
- Issue triage only: close resolved issues, consolidate duplicates, update labels
- No dead code removal, no Xcode warning cleanup, no hygiene pass

### Claude's Discretion
- Specific implementation approach for #37 fixes (evaluate proposed fixes vs alternatives)
- LiveActivityManager state machine design details
- Order of fix implementation (which cluster first)
- Log instrumentation to add for verification
