This task requires a physical device and Xcode Instruments. Claude cannot perform this.

**Setup:**
1. Connect iPhone 15 Pro Max to Mac via USB
2. Open Dictus.xcodeproj in Xcode
3. Select the DictusKeyboard scheme, target the physical device
4. Launch Instruments: Product > Profile (Cmd+I)
5. Choose "Blank" template, then add 3 instruments:
   - Allocations (for memory footprint)
   - Leaks (for retain cycle detection)
   - os_signpost (for KeyTapSignposter data -- filter by subsystem "com.pivi.dictus.keyboard", category "KeyPress")

**Profiling Protocol (record each operation for ~30 seconds):**

1. **Idle baseline:** Open any text field, activate Dictus keyboard, wait 30s. Note steady-state memory.

2. **Rapid typing:** Type rapidly for 30s (mix of letters, backspace, shift). Note peak memory.

3. **Dictation:** Tap mic, record for 30s, let transcription complete. Note peak during recording AND during transcription.

4. **Text prediction:** Type words slowly, observe suggestion bar updating. Note memory with predictions active for 30s.

5. **Emoji picker:** Open emoji picker, browse categories, search for "coeur". Note peak memory.

6. **Combined peak scenario:** Type rapidly for 10s, open emoji picker, close it, start dictation, let it transcribe, then type again. Note absolute peak.

**For each operation, record:**
- Peak memory (Allocations > All Heap & Anonymous VM > Dirty Size)
- Steady-state memory
- Number of leaks detected

**For signposter verification (during the rapid typing step):**
- Check os_signpost instrument for intervals labeled "touchDown" and "touchUp"
- Filter by subsystem "com.pivi.dictus.keyboard"
- Note whether intervals appear (yes/no) and approximate median duration

**Note about Instruments overhead:** Instruments disables the jetsam memory limit when attached, so readings may be 10-15MB higher than real-world. Also take a quick Debug Navigator baseline (Debug > View Debugging > Memory) without Instruments for comparison.
