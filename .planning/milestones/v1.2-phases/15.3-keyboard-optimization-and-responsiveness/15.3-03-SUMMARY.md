# Phase 15.3 Plan 03: Key Dimensions and Press Feedback Summary

**Device-adaptive KeyMetrics across 3 device classes with press feedback colors on delete/space/return/globe/emoji/layer switch keys**

## Accomplishments
- DeviceClass enum with compact/standard/large driving all KeyMetrics values
- Side margins (rowHorizontalPadding) now approximately equal to inter-key gap (keySpacing) -- fixes issue #29
- Press feedback colors on all non-letter special keys via pressedKeyColor and SpecialKeyButtonStyle

## Task Commits

Each task was committed atomically:

1. **Task 1: DeviceClass enum and device-adaptive KeyMetrics overhaul** - `e00a70c` (feat)
2. **Task 2: Press feedback colors on delete/space/return keys** - `427d791` (feat)

## Files Created/Modified
- `DictusKeyboard/Views/KeyButton.swift` - Added DeviceClass enum; refactored KeyMetrics with device-adaptive values and pressedKeyColor
- `DictusKeyboard/Views/SpecialKeyButton.swift` - Added SpecialKeyButtonStyle; press feedback on DeleteKey, SpaceKey, ReturnKey, GlobeKey, EmojiKey, LayerSwitchKey

## Decisions Made
- 3 device classes based on screen height breakpoints: 667pt (compact/SE), 852pt (standard), >852pt (large/Plus/Max)
- rowHorizontalPadding increased to match keySpacing per issue #29 finding (Apple keyboard side margins ~= inter-key gap)
- SpecialKeyButtonStyle as shared ButtonStyle for Button-based non-letter keys; DeleteKey and SpaceKey use their existing @State (isHolding/isPressed) for direct color toggle
- ShiftKey left unchanged -- icon state changes (empty/filled/capslock) are its press feedback per locked decision
- keyCornerRadius reduced from 6pt to 5pt to match Apple reference measurements
