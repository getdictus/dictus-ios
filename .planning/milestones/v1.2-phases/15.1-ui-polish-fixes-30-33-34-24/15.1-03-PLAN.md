**SoundFeedbackService.swift — Fix Bundle path (UAT tests 6/7/8):**

The Sounds folder is a folder reference in Xcode, so WAV files are at `AppBundle/Sounds/*.wav`, not at the bundle root. `Bundle.main.url(forResource:withExtension:)` returns nil because it only searches the root.

1. In the `play(_:)` method, change line 63:
   ```swift
   // BEFORE:
   guard let url = Bundle.main.url(forResource: soundName, withExtension: "wav") else {
       return
   }
   // AFTER:
   guard let url = Bundle.main.url(forResource: soundName, withExtension: "wav", subdirectory: "Sounds") else {
       PersistentLog.log("[Sound] WAV not found in bundle: \(soundName).wav (subdirectory: Sounds)")
       return
   }
   ```

2. Also add a debug log in the empty-name guard at the top of `play(_:)`:
   ```swift
   guard !soundName.isEmpty else {
       PersistentLog.log("[Sound] play() called with empty name")
       return
   }
   ```

3. Add a success log after the play call at the end of the method:
   ```swift
   cachedSounds[soundName] = soundID
   AudioServicesPlaySystemSound(soundID)
   PersistentLog.log("[Sound] Playing: \(soundName)")
   ```

**SoundSettingsView.swift — Fix picker layout (UAT test 5):**

The Picker is missing `.pickerStyle(.menu)`. SwiftUI defaults to `.automatic` which renders stacked (label above picker) inside a List. Adding `.menu` forces inline display (label left, menu right).

1. Add `.pickerStyle(.menu)` to the Picker in `soundPickerRow()`, right after the closing brace of ForEach:
   ```swift
   Picker(label, selection: selection) {
       ForEach(SoundFeedbackService.availableSounds(), id: \.self) { name in
           Text(SoundFeedbackService.displayName(for: name))
               .tag(name)
       }
   }
   .pickerStyle(.menu)
   ```

2. Remove the `Spacer()` between the Picker and the Button — `.menu` picker style handles inline layout automatically. The Spacer would push the preview button off-screen.

3. Fix the French accent: change `"Debut d'enregistrement"` to `"Début d'enregistrement"` (missing accent on 'e').

**RecordingOverlay.swift — Fix PillButton position offset (UAT test 3):**

The AnimatedMicButton in pill mode has a ring frame of 66x46pt. The 56x36pt pill sits centered inside, creating a 5pt inset from the leading edge. RecordingOverlay's PillButton has no ring, so with `.padding(.horizontal, 12)` it sits flush — 5pt to the left of where the mic button's pill appears.

Vertical offset: PillButton `.padding(.vertical, 8)` gives total height 36+16=52pt, centering the button at 26pt. The toolbar mic centers at 24pt. So the pill is 2pt lower.

Fix the padding on the button HStack in **recordingContent** (the recording state):

1. Change `.padding(.horizontal, 12)` to `.padding(.leading, 12)` and `.padding(.trailing, 17)` — the extra 5pt on trailing matches the ring inset of AnimatedMicButton, aligning the checkmark pill with the mic pill.

2. Change `.padding(.vertical, 8)` to `.padding(.vertical, 6)` — reduces total row height from 52pt to 48pt, moving button center from 26pt to 24pt (matching toolbar).

Apply the SAME padding changes to the **requestedContent** state's HStack (cancel-only row) for consistency.

Also update the **transcribingContent** reserved top bar to match: the `Color.clear.frame(height: 36)` with `.padding(.vertical, 8)` should become `.padding(.vertical, 6)` to keep the waveform Y-position stable across all three states.

Do NOT change the `.padding(.horizontal, 12)` on transcribingContent's Color.clear — it has no visible element so horizontal doesn't matter.
