    **Button alignment (#30) — RecordingOverlay.swift:**

    The mic button in ToolbarView is the position reference (56x36pt pill, .padding(.horizontal, 12)). The RecordingOverlay buttons must match exactly so the overlay fade-in looks seamless.

    1. Change PillButton frame from `height: 44` to `height: 36` (line ~240: `.frame(width: 56, height: 44)` becomes `.frame(width: 56, height: 36)`)
    2. Change all `.padding(.horizontal, 16)` inside recording/transcribing content HStacks to `.padding(.horizontal, 12)` to match ToolbarView's horizontal padding. There are 3 occurrences: lines ~90, ~135, ~184.
    3. Change the transcribing content reserved height from `.frame(height: 44)` to `.frame(height: 36)` (line ~183) to prevent vertical jump during recording-to-transcribing transition.
    4. Keep .padding(.vertical, 8) unchanged — it provides comfortable vertical spacing.
    5. Both cancel (xmark) and check (checkmark) PillButtons use the same PillButton struct so both get 56x36 automatically.

    **Swipe-back overlay (#34) — SwipeBackOverlayView.swift:**

    1. Add Dynamic Island capsule to the iPhone mockup in SwipeAnimationView. Inside the ZStack that draws the phone frame, add a VStack at the top:
       ```swift
       VStack {
           RoundedRectangle(cornerRadius: 10)
               .fill(Color.white.opacity(0.25))
               .frame(width: 38, height: 11)
               .padding(.top, 14)
           Spacer()
       }
       ```
       Place this BEFORE the existing home indicator content in the ZStack.
       Proportions: real iPhone 15 Pro Dynamic Island is ~32% of screen width. Scaled to 120pt mockup = ~38pt wide, ~11pt tall, ~14pt from top.

    2. Change animation duration from 1.5 to 0.8: find `.easeInOut(duration: 1.5)` and replace with `.easeInOut(duration: 0.8)`.
