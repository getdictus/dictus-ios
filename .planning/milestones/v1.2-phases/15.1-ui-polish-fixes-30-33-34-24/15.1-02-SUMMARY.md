**AudioToolbox-based recording sound feedback with SoundSettingsView (toggle + 3 pickers + preview) and DictationCoordinator lifecycle integration**

## Accomplishments
- SoundFeedbackService plays WAV sounds via AudioServicesPlaySystemSound with cached SystemSoundIDs
- Start sound plays before WhisperKit audio session configuration; stop/cancel sounds play after session release
- SoundSettingsView provides global toggle + 3 event pickers with inline preview buttons
- Sound preferences persist via App Group SharedKeys, readable by keyboard extension
- 29 WAV files bundled as folder reference in DictusApp resources

## Files Created/Modified

## Decisions Made
- Used AudioServicesPlaySystemSound (not AVAudioPlayer) because it respects the physical silent switch natively
- Start sound played synchronously before configureAudioSession() -- WhisperKit's session takeover would suppress playback if done after
- Hardcoded the 29 sound file names in availableSounds() instead of Bundle directory enumeration for reliability
- Sounds folder added as folder reference (not individual file references) for cleaner project structure
