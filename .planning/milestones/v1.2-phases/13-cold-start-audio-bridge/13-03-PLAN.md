**1. DictusApp.swift -- Add auto-return after cold start URL handling:**

In `handleIncomingURL`, after setting `coldStartActive` and calling `startDictation(fromURL: true)` (both added by Plan 01), add auto-return logic:

```swift
case "dictate":
    let isFromKeyboard = URLComponents(url: url, resolvingAgainstBaseURL: false)?
        .queryItems?.first(where: { $0.name == "source" })?.value == "keyboard"

    if isFromKeyboard {
        AppGroup.defaults.set(true, forKey: SharedKeys.coldStartActive)
        AppGroup.defaults.synchronize()
        DictusLogger.app.info("Cold start dictation from keyboard")
    }

    coordinator.startDictation(fromURL: true)

    // Auto-return: attempt to send user back to source app
    if isFromKeyboard {
        attemptAutoReturn()
    }
```

Add a private method `attemptAutoReturn()` that iterates ALL known app schemes:
```swift
private func attemptAutoReturn() {
    // Since keyboard cannot reliably detect the active source app,
    // iterate known app schemes and try to open the FIRST installed one.
    // This is a best-effort heuristic: it opens the first match,
    // which may not be the actual source app.
    // If no known app is installed, the swipe-back overlay handles it.
    for appScheme in KnownAppSchemes.all {
        guard let url = URL(string: appScheme.scheme) else { continue }
        if UIApplication.shared.canOpenURL(url) {
            DictusLogger.app.info("Auto-returning to \(appScheme.name) via \(appScheme.scheme)")
            UIApplication.shared.open(url)
            return
        }
    }
    DictusLogger.app.info("No known app installed for auto-return, showing swipe-back overlay")
}
```

IMPORTANT DESIGN NOTE: This heuristic tries installed apps in order (WhatsApp first, then Messages, Telegram, etc.). It will return to the FIRST installed known app, which may not be the app the user was actually typing in. This is acknowledged as a v1.2 limitation. For v1.3, a more sophisticated approach could be explored (e.g., keyboard writing the most recently focused app to App Group). The swipe-back overlay remains the reliable fallback for cases where the heuristic picks the wrong app.

WHY iterate instead of reading sourceAppScheme: The keyboard's `detectAndSaveSourceApp()` always writes "unknown" because there is no public API to detect the active host app from a keyboard extension. Rather than having a dead code path that checks "unknown" and exits, the app takes the pragmatic approach of trying known schemes directly.

WHY auto-return AFTER startDictation: Per research pitfall #3, audio session must be activated before switching apps. startDictation activates the session synchronously. Auto-return happens after.
