   WHY URL parameter: The keyboard explicitly tells the app "I opened you" -- this is the source of truth. The App Group flag persists it so MainTabView can read it.

   The URL parsing approach: `URLComponents(url: url, resolvingAgainstBaseURL: false)?.queryItems?.first(where: { $0.name == "source" })?.value == "keyboard"`.

2. **DictusApp.swift -- scenePhase cleanup:**
   In the existing `.onChange(of: scenePhase)` handler, add cleanup for the `case .background:` branch:
   ```swift
   AppGroup.defaults.set(false, forKey: SharedKeys.coldStartActive)
   AppGroup.defaults.removeObject(forKey: SharedKeys.sourceAppScheme)
   AppGroup.defaults.synchronize()
   ```
   WHY: Prevents stale cold start state from persisting. When the user leaves the app (background), the overlay should not show on next normal open.

3. **MainTabView.swift -- conditional rendering:**
   Add `@State private var isColdStartMode = false` to MainTabView. In the body, wrap the existing content:
   ```swift
   var body: some View {
       ZStack {
           if isColdStartMode {
               // Placeholder for SwipeBackOverlayView (Plan 02 will replace)
               ZStack {
                   LinearGradient(

                       startPoint: .topLeading,
                       endPoint: .bottomTrailing
                   )
                   .ignoresSafeArea()
                   Text("Swipe back to keyboard")
                       .foregroundColor(.white)
               }
           } else {
               // Existing TabView content (keep exactly as-is)
           }
       }
       .onOpenURL { url in
           if let host = url.host, host == "dictate",
              let query = URLComponents(url: url, resolvingAgainstBaseURL: false)?.queryItems,
              query.contains(where: { $0.name == "source" && $0.value == "keyboard" }) {
               isColdStartMode = true
           }
   ```
