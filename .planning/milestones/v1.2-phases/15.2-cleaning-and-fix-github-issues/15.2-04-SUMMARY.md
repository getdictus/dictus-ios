**BrandWaveform forced recreation via .id(waveformRefreshID) on keyboard reappear, with displayLevels seeded from live energy data for instant first-frame rendering**

## Accomplishments
- waveformRefreshID counter in KeyboardState increments on every keyboard reappear, forcing SwiftUI to destroy and recreate all 3 BrandWaveform instances
- BrandWaveform seeds displayLevels from current energyLevels via targetLevels() on .onAppear, eliminating the ~10 frame animation catch-up delay
- No destructive reset on keyboardAppear -- Phase 12 constraint preserved (refreshFromDefaults + watchdog pattern intact)
