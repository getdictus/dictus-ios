    - The old PersistentLog.log(_ message: String) is removed or made internal/private (no public free-text API)
    - maxLines is 500 (up from 200) (LOG-04)
    - trimIfNeeded keeps exactly the last 500 lines when file exceeds 500
    - File writes use NSFileCoordinator for cross-process safety
    - File reads use NSFileCoordinator for cross-process safety
    - PersistentLog.exportContent() returns device header + full log content
    - Device header format: "Dictus Debug Log\niOS {version} | App {version} ({build}) | {deviceModel} | Model: {activeModel}\n---\n"
    - read() still returns raw log content (for DebugLogView)
    - clear() still works
  </behavior>
  <action>
    Evolve the existing PersistentLog.swift:

    1. Change `maxLines` from 200 to 500.
    2. Add new public method `log(_ event: LogEvent)` that calls `event.formatted()` and appends to file. This is the primary public API.
    3. Make the old `log(_ message: String, function:)` method `internal` (not public) so external callers must use the typed API. Keep it temporarily for any internal migration needs, but mark with `@available(*, deprecated, message: "Use log(_ event: LogEvent) instead")`.
    4. Replace raw `DispatchQueue.global().async` + `FileHandle` writes with `NSFileCoordinator` pattern from research. Use `coordinator.coordinate(writingItemAt:options:.forMerging)` for writes. Keep the serial DispatchQueue for internal ordering within a single process, but the coordinator handles cross-process safety.
    5. Also wrap `read()` and `trimIfNeeded()` in NSFileCoordinator for read safety.
    6. Add `exportContent() -> String` that assembles the device header using UIDevice.current.systemVersion, UIDevice.current.model, Bundle.main version/build, and SharedKeys.activeModel from AppGroup.defaults. Prepend to read() output.
    7. Forward log events to DictusLogger (os.log) for Xcode console visibility when debugger is attached. Map LogLevel to os.log levels.

    Note: UIDevice is only available in iOS, not in SPM test environment. Guard `exportContent()` with `#if canImport(UIKit)` and provide a testable internal helper that accepts header parameters.

    - Test that log(LogEvent) writes formatted output to a temp file
    - Test that trim keeps exactly 500 lines
    - Test that after writing 510 lines, only 500 remain
    - Test exportContent header format (use the testable helper with injected values)
